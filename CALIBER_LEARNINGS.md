# Caliber Learnings

Accumulated patterns and anti-patterns from development sessions.
Auto-managed by [caliber](https://github.com/caliber-ai-org/ai-setup) — do not edit manually.

- **[correction:personal]** Do NOT run multiple Agent/sub-agent tasks in parallel — run one at a time. **Why:** parallel sub-agents burn the extra-usage budget and hit rate limits mid-batch, and concurrent writes to shared files collide. **How to apply:** when doing multi-step work that would naturally fan out into sub-agents, sequence them — wait for each to finish before launching the next.
- **[correction:personal]** Keep SVN credentials in `.github/workflows/tests.yml` HARDCODED, do NOT refactor them to `${{ secrets.SVN_USER }}` / `${{ secrets.SVN_PASS }}`. **Why:** the secrets don't exist in the repo's GitHub settings yet, breaking CI silently. **How to apply:** don't proactively move CI credentials to `secrets.*` without being asked.
- **[correction:personal]** When invoking external CLI tools, pass ALL flags every time even when the value is empty — `--operator-user '' --operator-pass ''` rather than conditionally omitting the flag. **Why:** user explicitly reverted a conditional-flag implementation — keeping the CLI signature stable is preferred. **How to apply:** build commands with all expected flags using `escapeshellarg((string)($field ?? ''))`.
- **[fix:project]** PHPUnit 4.x/5.x/6.x crashes on PHP 8.x with `Call to undefined function each()` — `each()` was removed in PHP 8.0. **Why:** the original `composer.json` pinned `"phpunit/phpunit": "^4.8.35 | ^5.5 | ^6.5"` which installs 4.x on a clean install. **How to apply:** the project now requires `"phpunit/phpunit": "^9.6"`. If you ever reset `vendor/` and see this error, run `rm -rf vendor composer.lock && composer install` after confirming `composer.json` has the `^9.6` constraint.
- **[gotcha:project]** `bin/parse.php` translates Wikipedia's `N`→`#` and `A`→`@` globally, which corrupts format strings that contain literal country-code letter prefixes (e.g., `AI-2640` for Anguilla gets mangled to `@I-2640`, `BIOT 1ZZ` to `BB#D 1ZZ`). **How to apply:** after running `bin/parse.php` and pasting output into `src/Validator.php`, manually grep for `'@[A-Z]'` and literal-string entries like `'BIQQ 1ZZ'` and audit whether `@` and `#` represent wildcards or corrupted literals.
- **[gotcha:project]** Duplicate format strings appear silently in `$formats` entries after running `bin/parse.php` (e.g., `['####', '####', '@####@@@']` for Argentina, `['#####', '#####-###', '#####-###']` for Brazil). `isValid()` still works but the duplication wastes space and can confuse users of `getFormats()`. **How to apply:** after a Wikipedia update pass, grep for repeated adjacent values within each country's array and deduplicate.
- **[pattern:project]** PHPUnit 9 requires updated `phpunit.xml.dist` syntax. Replace the old `<filter>/<whitelist>` block with `<coverage>/<include>`, add `xmlns:xsi` and `xsi:noNamespaceSchemaLocation` to the root `<phpunit>` element, and remove the deprecated `syntaxCheck="false"` attribute. Without these changes PHPUnit 9 emits deprecation warnings and the config is not schema-valid.
- **[convention:project]** Format strings in `$formats` use only `#` (digit), `@` (letter), literal hyphens, spaces, and fixed alphanumeric characters. Any literal letters that are part of a fixed prefix (e.g., `BIOT`, `AI-`) must appear verbatim — do NOT use `@` for them. Only use `@` where any letter is valid.
